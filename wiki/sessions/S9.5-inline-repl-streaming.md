# S9.5 — Inline REPL + day-1 streaming

## Why this session existed

S9 shipped M2 as a Textual full-screen TUI. The first time I ran `TERN_LIVE=1 tern chat`
and saw it take over my terminal, I bounced. That's not the feel — Claude Code, aider,
and Hermes all live inline: prompt under the cursor, output streams above, scrollback
preserved. So we ripped Textual and rebuilt the chat surface.

While we were in there, we collapsed two roadmap items in:
- Streaming day-1 (was deferred)
- Diff-up-front in the edit_block approval prompt (was a `[d]` keystroke before)

## What shipped

### Streaming all the way down
- `LLMTextDelta` event in `core/events.py` (parent_id → matching `LLMRequested.id`).
- `StreamingProviderAdapter` Protocol in `core/provider.py`. Optional — the loop probes
  via `hasattr(adapter, "stream")` so fakes/offline adapters keep working.
- `BedrockAnthropicAdapter.stream()` driving `invoke_model_with_response_stream`.
  Stream tuple shape: `("text", str)` | `("tool_use_start", dict)` |
  `("tool_args_delta", str)` | `("done", ProviderResponse)`.
- `core/loop.py` consumes either path: when `stream()` is present it forwards `text`
  deltas as `LLMTextDelta` events and waits for the final `("done", ProviderResponse)`
  for usage/cost. `complete()` retained as the non-streaming fallback.
- `FakeStreamingAdapter` test double + `tests/test_loop_stream.py` — verifies
  ordering (`LLMRequested → deltas → LLMResponded → TurnCompleted`), parent_id
  linkage, and the empty-reply edge case.

### Inline REPL (`src/tern/ui/app.py`)
- `prompt_toolkit.PromptSession` for input — history, multiline (Esc, Enter), Ctrl+C
  semantics, `patch_stdout` so background renders don't fight the prompt cursor.
- `rich.live.Live` for the streaming assistant message; closes cleanly on every
  non-streaming event (tool calls, approvals) so scrollback stays linear.
- Tool calls render as one-liners: `→ read_file path=…`, `✓ read_file 3,054B`.
- `_StreamRenderer` owns the live region and is the only thing that touches stdout
  during a turn. Keeps the rendering invariants in one place.
- Multi-turn history threaded through `history: list[CanonicalMessage]`; each turn
  picks up the previous assistant message via `adapter.last_response_message`.

### Permission prompt (inline, diff up-front)
- `_build_inline_prompter` returns a `Prompter` callable wired into `PermissionGate`.
- For `edit_block`, the prompt prints a unified diff in a Rich `Panel(Syntax(...))`
  *before* the y/N question — no extra keystroke needed to see what changes.
- Falls back to a `--- search +++ replace` shape when `difflib.unified_diff` returns
  empty (single-line edit), and warns when the `search` block isn't in the file.
- Y/N read via `PromptSession.prompt("approve? [y/N] ")` so it composes with
  prompt_toolkit's input system (no terminal-mode collisions).

### Ctrl+C semantics
- Inside `asyncio.run(_go())`: KeyboardInterrupt is caught, the live region is closed,
  we print `· turn cancelled (Ctrl+C again to exit)`, then we re-prompt. A second
  Ctrl+C at the prompt exits the REPL cleanly.

### Dependency swap
- Dropped `textual>=0.80` from `pyproject.toml`.
- Added `prompt_toolkit>=3.0`.
- mypy override switched from `textual.*` to `prompt_toolkit, prompt_toolkit.*`.

## Pitfalls hit + the fixes

1. **Loop variable closures inside the REPL.** The `async def _go()` defined inside
   the `while True:` loop captured `turn`/`adapter`/`renderer` by reference, which
   ruff B023'd. Fix: bind them as default args (`_turn: Turn = turn, …`) so each
   coroutine snapshots its own values. Also makes mypy --strict happy with explicit
   types on the bound defaults.

2. **mypy: `adapter.stream` unknown.** Pyright/mypy can't see optional attributes
   gated by `hasattr`. I left `# type: ignore[attr-defined]` originally, then dropped
   it once `ProviderAdapter` was widened — currently the loop calls `adapter.stream()`
   without a guard ignore because mypy is satisfied by the runtime `hasattr` check
   plus the union-typed call site. (If this regresses in a future mypy bump, re-add
   the ignore on that single line — don't re-architect.)

3. **`patch_stdout` is required when prompts coexist with Rich output.** Without it,
   prompt_toolkit's input line gets corrupted by any concurrent stderr write. Both
   the main REPL prompt and the approval prompt wrap their `prompt(...)` call in
   `with patch_stdout():`.

4. **Diff preview when `search` is missing.** edit_block's fuzzy matcher will still
   try, but the user deserves a heads-up. Diff renderer prepends a `# warning:` line
   so the operator sees it before saying y.

5. **Streaming smoke: AWS env not in subshell.** Same pitfall logged in S8 — `tern run`
   needs `AWS_ACCESS_KEY_ID/SECRET/REGION` in the current shell. Use
   `eval "$(grep -E '^export AWS_' ~/.zshrc)"` before `TERN_LIVE=1 tern run …`.
